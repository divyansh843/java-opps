# OOPS in Java
OOP = Object Oriented Programming
Java ki poori backbone OOP hai.
OOP ek programming style hai jisme hum real-life चीजों को code की form में model करते हैं.

# Why do we need OOP?
1️⃣ Code Reusability
Ek parent class bana ke usse child classes use kar sakti hain.

2️⃣ Data Security
Private data + getters/setters → secure data.

3️⃣ Maintainability
Problem solve karna easy.

4️⃣ Faster Development
Team me alag-alag log alag classes par kaam kar sakte hain.

# OOP ka Working Style
OOP ke andar sab kuch revolve karta hai:
Class (design)
Object (real entity)
Constructor (object initialise)
Pillars (Encapsulation, Inheritance, Polymorphism, Abstraction)

# What is an Object?
Object = class ka instance
Object memory me real exist karta hai.

# What is a Constructor?
A. Definition:
Constructor ek special method hota hai jo object ko initialize karta hai.

Initialize = Object ke variables ki initial values set karna.

B. Important Properties of Constructor
Constructor ka naam class ke naam jaisa hota hai
Constructor ka koi return type nahi hota
Object create hote hi automatically call hota hai
new keyword ke saath work karta hai

C. Types of Constructors
Java me 3 types of constructors hote hain:

Default Constructor (Compiler generated)
Agar tum class me constructor nahi likhte,
Java compiler ek default constructor banata hai.

No-Argument / User-defined Default Constructor
